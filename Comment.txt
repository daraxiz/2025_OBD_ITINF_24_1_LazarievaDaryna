1. Загальна інформація

Даний файл містить інструкцію щодо розгортання програмного продукту (веб-застосунку) курсової роботи з дисципліни «Основи баз даних (ОБД)», а також короткий опис користувачів системи та їх ролей.

Проєкт являє собою веб-інформаційну систему для перегляду та управління онлайн-манґою, реалізовану на платформі ASP.NET Core MVC з використанням СУБД Microsoft SQL Server.


2. Вимоги до програмного забезпечення

Для коректного розгортання та запуску застосунку необхідно:

2.1. Операційна система
- Windows 10 / 11 (рекомендовано).

2.2. Сервер баз даних
- Microsoft SQL Server (LocalDB / Express / Developer)  
  або сумісна версія SQL Server.
- SQL Server Management Studio (SSMS) для роботи з базою даних.

2.3. Платформа .NET
- Встановлений .NET SDK (наприклад .NET 8.0 або версія, вказана у файлі .csproj).

2.4. Середовище розробки (будь-який варіант)
- Visual Studio 2022 / JetBrains Rider / VS Code (з підтримкою C#)  
  — рекомендовано Visual Studio 2022.

2.5. Браузер
- Сучасний браузер (Google Chrome, Microsoft Edge тощо).


3. Структура проєкту (загально)

В архіві/репозиторії проєкту містяться, зокрема, такі важливі папки:

- Mankura.sln` — рішення Visual Studio.
- Mankura/` — основний веб-проєкт (ASP.NET Core MVC):
  - `Controllers/` — контролери.
  - `Models/` — моделі домену та ViewModel-и.
  - `Data/` — класи доступу до БД (репозиторії, Db helper).
  - `Views/` — Razor-представлення (інтерфейс користувача).
  - `wwwroot/` — статичні файли (CSS, JS, зображення).
  - `appsettings.json` — налаштування, включно з рядком підключення до БД.
- `DB/` — база даних:
  - файл(и) *.mdf — основний файл бази даних;
  - файл(и) *.ldf — файл журналу транзакцій.

Назви файлів бази даних та їх розташування можуть незначно відрізнятися — важливо вказати коректний шлях при підключенні.


4. Підготовка бази даних (Attach MDF/ LDF)

4.1. Запуск SQL Server Management Studio (SSMS)
- Відкрити SSMS.
- Підключитися до локального екземпляра SQL Server, наприклад:
  - `(localdb)\MSSQLLocalDB` або
  - `.\SQLEXPRESS` або інший, який використовується на ПК.

4.2. Приєднання (Attach) бази даних
1) У вікні **Object Explorer**:
   - Правою кнопкою миші по вузлу **Databases** → **Attach...**.

2) У діалоговому вікні **Attach Databases**:
   - Натиснути **Add...**.
   - Вибрати файл бази даних формату `*.mdf` з папки `DB` (наприклад, `MankuraDB.mdf` або інша назва).

3) Після вибору `.mdf`:
   - Переконатися, що відповідний `.ldf` файл (файл журналу) знайдений автоматично.  
     Якщо ні — вказати шлях до нього вручну або підтвердити, якщо SQL Server дозволяє створити новий журнал.

4) Натиснути **OK** для завершення підключення.

4.3. Перевірка
- У списку **Databases** має з’явитися приєднана база даних (назва — наприклад: `MankuraDB`).
- Запам’ятати точну назву бази (Database Name) — вона буде потрібна при налаштуванні рядка підключення.


5. Налаштування рядка підключення (appsettings.json)

5.1. Відкриття файлу
- У папці `Mankura/` відкрити файл `appsettings.json`.

5.2. Пошук секції `ConnectionStrings`
У файлі буде секція приблизно такого вигляду:

```json
"ConnectionStrings": {
  "DefaultConnection": "Server=(localdb)\\MSSQLLocalDB;Database=MankuraDB;Trusted_Connection=True;MultipleActiveResultSets=true"
}

Після запуску застосунку доступні такі частини системи:

Головна сторінка / каталог манґ.

Сторінка деталей манґи (з описом, інформацією, розділами та коментарями).

Читання розділів (перегляд сторінок).

Панель користувача (якщо реалізована).

Панель адміністратора (/Admin/Panel) — доступна лише користувачам з роллю Admin.

6. Типи користувачів

Гість (неавторизований користувач)

Може переглядати каталог манґ, відкривати сторінки з описом, дивитися розділи (якщо дозволено).

Не може:

залишати коментарі;

додавати манґу до списку читання;

користуватися панеллю адміністратора.

Зареєстрований користувач (User)

Має акаунт у системі (логін/пароль).

Може:

авторизуватися;

залишати коментарі до манґ;

додавати манґу до списку читання (різні статуси: буде читатися, читається, прочитано, улюблене);

переглядати власний список читання.

Адміністратор (Admin)

Має повний доступ до панелі керування:

додавання нової манґи;

редагування інформації про манґу (назва, опис, обкладинка, статус, тип, рік, жанри тощо);

додавання нових розділів;

імпорт сторінок глав;

редагування або видалення манґи, розділів та коментарів;

керування користувачами (зміна ролей, видалення користувачів тощо).
