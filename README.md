Коротка характеристика (реферат):
У роботі розроблено інформаційну систему для керування каталогом манґи та процесом читання користувачами. 
Система дозволяє переглядати інформацію про манґу, додавати та редагувати глави, працювати з коментарями, 
формувати список для читання, а також здійснювати адміністрування контенту. 
База даних забезпечує збереження користувачів, манґи, жанрів, глав, сторінок та історії читання. 
Рішення реалізовано як веб-застосунок з використанням архітектури клієнт-сервер.

Entity Framework / ADO.NET: Керується через NuGet 6.14

Коментар щодо запуску проєкту:
1) Підготовка бази даних
   - У папці `DB` знаходяться файли бази даних у форматах `*.mdf` та `*.ldf`.  
   - Відкрити **SQL Server Management Studio (SSMS)**.  
   - Підключитися до локального SQL Server (наприклад: `(localdb)\MSSQLLocalDB` або свій екземпляр).  
   - У Object Explorer обрати:
     - правою кнопкою на вузол **Databases** → **Attach...**  
     - у вікні **Attach Database** натиснути **Add...**, обрати файл `*.mdf` з папки `DB`.  
     - Підтвердити підключення (ОК).  
   - Перевірити, що база з’явилася у списку баз даних (назву БД за потреби можна використати в connection string).
2) Налаштування рядка підключення

   - Відкрити файл `appsettings.json` у корені веб-проєкту.  
   - Знайти секцію:
     `"ConnectionStrings": { "DefaultConnection": "..." }`
   - Замінити значення `DefaultConnection` на актуальний рядок підключення до вашого SQL Server та підключеної БД, наприклад:
     `Server=(localdb)\\MSSQLLocalDB;Database=Назва_Твоєї_БД;Trusted_Connection=True;MultipleActiveResultSets=true;`
   - За необхідності врахувати:
     - логін/пароль SQL Server (якщо використовується SQL auth),
     - іншу назву сервера чи бази.
3) Запуск застосунку
   Варіант 1 — через Visual Studio:
   - Відкрити `.sln` файл проєкту з папки `program`.  
   - Обрати конфігурацію `Debug` та потрібний цільовий проєкт (web-проєкт).  
   - Переконатися, що обраний проєкт позначений як **Startup Project**.  
   - Натиснути **Run / IIS Express / ▶** для запуску.  
   - Після запуску застосунок буде доступний за адресою типу:  
     `https://localhost:7102/` (порт може відрізнятись — дивитися в адресному рядку).
   Варіант 2 — через .NET CLI:
   - Відкрити термінал/консоль у папці з web-проєктом (там, де файл `.csproj`).  
   - Виконати команду:
     `dotnet restore`  
     `dotnet run`
   - Дочекатися повідомлення про те, що застосунок запущено, і перейти за вказаною адресою (наприклад, `https://localhost:7102`).
4) Авторизація та перевірка ролей

   - Зайти під обліковим записом **Admin**, вказаним у розділі g.  
   - Перейти у панель адміністратора (`/Admin/Panel` або зі сторінки меню, якщо є посилання).  
   - Перевірити роботу:
     - додавання/редагування/видалення манґи;
     - додавання глав та імпорту сторінок;
     - редагування користувачів і їхніх ролей.

   - Зайти під обліковим записом звичайного користувача та перевірити:
     - додавання в список читання;
     - коментування манґи;
     - перегляд розділів.
